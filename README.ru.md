# Claude Booster

**Хватит каждое утро заново учить Claude Code одному и тому же.**

Claude Code «из коробки» не помнит ничего между сессиями, не накапливает институциональный опыт и не переносит знания между проектами. К третьей неделе ежедневной работы ты замечаешь:

- Ты заново объясняешь стек, конвенции, режимы отказа — каждую сессию.
- Claude переписывает хелпер, который у тебя уже есть, потому что не сделал grep.
- Те же уточняющие вопросы день за днём. («npm или pnpm?» — ты ответил вчера.)
- Хук тихо перестал срабатывать, а обнаружил ты это через 3 недели.
- Каждый новый проект стартует с нуля. Добытые потом уроки со старого проекта не переносятся.

Claude Booster превращает эти сессии в накопительный актив. Одна команда `python install.py` на любом Mac или Linux — и твой Claude Code начинает **помнить, учиться и аудитить сам себя**.

## 🚀 Начни отсюда (гайд для новичка)

Впервые видишь Claude Booster? Читать весь changelog не нужно. Вот версия на 5 минут.

### 1. Установка (одна команда)

```bash
git clone https://github.com/demetrius2017/claude-booster.git
cd claude-booster && python install.py
```

Всё. Booster ставит набор правил, хуков и slash-команд в `~/.claude/`. Твоя следующая сессия Claude Code уже ведёт себя иначе — читает собственную память на старте и заставляет себя проходить пайплайн качества.

### 2. Твоя первая сессия

| Шаг | Набери | Что происходит |
|-----|--------|----------------|
| Открыть сессию | `/start` | Claude читает вчерашний handover, базу знаний и телеметрию здоровья, затем предлагает план. **Больше не нужно заново объяснять стек каждое утро.** |
| Писать код | `/go <что нужно сделать>` | Задача проходит через гейтовый кросс-провайдер пайплайн (дизайн → челлендж → сборка → независимый тест → ревью диффа). Вердикт — **exit code**, а не «выглядит норм». |
| Закрыть сессию | `/handover` | Сохраняется структурный отчёт, чтобы завтрашний `/start` продолжил ровно с того места, где ты остановился. |

### 3. Каталог команд — что делает и какую проблему решает

**Ежедневный цикл — память между сессиями**

| Команда | Что делает | Какую проблему убивает |
|---------|------------|------------------------|
| `/start` | Восстанавливает контекст: README, последний handover, кросс-проектную базу знаний, телеметрию — и планирует | Claude стартует каждую сессию вслепую, без памяти о том, что было вчера |
| `/handover` | Структурный отчёт конца сессии (git log, Goal+KPI, required reading, evidence) | Сессия закончилась → открытые нити и добытый контекст испаряются |
| `/debt` | Отслеживает и закрывает недоделки, которые переживают ресет сессии | «Потом поправлю» забывается в момент закрытия сессии |

**Пайплайн качества — ключевая ценность: ни одна модель не проверяет свою работу**

| Команда | Что делает | Какую проблему убивает |
|---------|------------|------------------------|
| `/go` | **Семёрка** (7 стадий): Flow Designer → Challenge → Prototype Gate → Worker + Verifier → Test → Diff-review → Verdict. Кросс-провайдер, без обхода, вердикт по exit code | Одноагентный код сам ставит себе зачёт и мыслит «плоскими снимками» (верно в T=0, неверно в T+1) |
| `/audit` | Шесть параллельных агентов-линз (корректность, безопасность, производительность, архитектура, целостность данных, эксплуатация) + внешнее ревью | Один ревьюер упускает то, что шесть специалистов ловят независимо |
| `/code-review` | Точечный проход после правки: дубли, оверинжиниринг, дрейф интеграции, неэффективность | Быстрые ревью упускают класс «ты уже написал этот хелпер» |
| `/consilium` | Дебаты 3–5 ролевых агентов + внешние модели, синтез в отчёт | Рискованные архитектурные решения требуют состязательных перспектив, а не одного мнения |
| `/hackathon` | N воркеров строят одну фичу в изоляции; Судья тестит всех одним сьютом; побеждает лучший счёт | Для по-настоящему неопределённых решений одна попытка — это подброс монеты |

**Верификация — докажи, что работает, не предполагай**

| Команда | Что делает | Какую проблему убивает |
|---------|------------|------------------------|
| `/verify-after-edit` | Прогоняет реальный UI-флоу после правки, а не только тесты | «Компилируется» ≠ «работает» |
| `/verify-flow` | Проверяет полный UI-флоу end-to-end | Отдельные фиксы проходят, а флоу, в котором они живут, сломан |
| `/audit-trace` | Трассирует один концепт по всем путям вычислений; расхождение = дефект | Тихая порча данных, где два пути кода считают «одно и то же» по-разному |

**Системное мышление и инфра — пойми радиус поражения до правки**

| Команда | Что делает | Какую проблему убивает |
|---------|------------|------------------------|
| `/architecture` | Генерит «печатную плату» — `ARCHITECTURE.md` + `dep_manifest.json` — что с чем связано | Claude правит функцию A и молча ломает B, C, D, которые от неё зависели |
| `/phase` | Показывает/ставит фазу воркфлоу (RECON → PLAN → IMPLEMENT → AUDIT → VERIFY → MERGE) | Правку до понимания; гейт фазы требует «диагноз до починки» |
| `/delegate` | Инспектирует/контролирует гейт делегирования (Lead должен делегировать код, а не писать сам) | Оркестратор тихо делает работу сам, вместо использования нужной модели |
| `/lead` | Ведёт supervised-воркера (квота + детектор тишины) | Долгие автономные задачи дрейфуют или зависают без надзора |
| `/fable` | Разовое read-only второе мнение от Fable 5 (не полный консилиум) | Нужна одна проверка-санитайзер, а не пять агентов и отчёт |
| `/update` | Автообновление Booster с GitHub, не выходя из сессии | Отставание от фиксов, потому что апгрейд — это трение |

### 4. Почему это иначе, в одну фразу

Три механизма, каждый бьёт по своему способу, которым LLM-агенты проваливаются на мультисессионных проектах:

1. **Темпорально-причинная память** — хранит *причинные цепочки* (пробовал → случилось → заключил → осталось открытым), а не просто факты. Убивает цикл «заново нахожу тот же баг каждую неделю».
2. **Пайплайн Семёрка (`/go`)** — сильная модель *думает* (критика дизайна, независимая верификация, ревью диффа), быстрая flat-fee модель *печатает*. **Ни одна модель не ревьюит свой код**, а вердикт — exit code зелёного теста, а не ощущение.
3. **Умный роутинг моделей** — Haiku для поиска, Sonnet для кода, Opus для сложных рассуждений, flat-fee Codex где эквивалентно. Правильная модель, правильная задача, правильная стоимость.

> **Новичку:** запусти `python install.py`, открой сессию через `/start`, собери что-нибудь через `/go` и закрой через `/handover`. Всё остальное — глубина, к которой потянешься, когда понадобится.
