# Post-Edit UI Verification

Автоматическая проверка UI после правок фронтенд-кода.

## Протокол

### 1. Определи что изменилось
- Прочитай git diff (staged + unstaged) для фронтенд-файлов (.tsx, .jsx, .vue, .svelte, .css, .html)
- Определи какие страницы/компоненты затронуты
- Найди URL этих страниц (из роутера проекта или package.json dev server)

### 2. Для КАЖДОЙ затронутой страницы запусти /verify-flow
- Передай URL страницы
- Фокусируйся на изменённых компонентах

### 3. Regression check
- Проверь что соседние элементы не сломались
- Console должна быть чистой (новые ошибки = FAIL)
- Network requests не должны возвращать ошибки

### 4. Итог
Сводная таблица по всем проверенным страницам:

| Страница | Статус | Проблемы |
|----------|--------|----------|
| /dashboard | PASS | — |
| /settings | FAIL | таблица пустая, API 500 |

Если всё PASS — продолжай работу.
Если FAIL — покажи проблемы и исправь, затем перепроверь.
