## What was done

### 1. supervisor.py — orchestration loop + CLI — 382 LOC
`templates/scripts/supervisor/supervisor.py`.

- `Supervisor` class wires `StreamJsonRuntime → policy.evaluate →
  quota.admit → detector.on_event → persistence.record_decision` into
  a single async loop.
- **Every tool event resolves to an authoritative `approve` or
  `deny`** (audit-fix C1). No more "log-and-hope" on deny. On any
  non-approve verdict the supervisor calls `runtime.cancel(task_id)`
  and forces the detector to `CANCELLED`. A worker process mid-flight
  on a denied tool gets SIGINT→SIGTERM→SIGKILL immediately.
- **No escalator configured + `escalate` policy verdict → default-deny +
  cancel** (audit-fix C1). The pre-fix code silently fell through to
  log-only, which the Haiku prompt implied was authoritative. Not any
  more.
- **Silence poller acts on `POSSIBLY_COMPLETE`** (audit-fix H2). Hung
  workers get the same cancel ladder as denied tools. Previously the
  poller computed the state and threw it away.
- **Loop-guard filters prior approvals only** (audit-fix M4). 3 prior
  denies of the same `(tool, args)` no longer incorrectly trip
  loop-guard on a legitimate 4th approve.
- **`--cwd` drives both `project_dir` AND default `.claude/
  supervisor.yaml` lookup** (audit-fix M3). Previously running from
  repo A with `--cwd=B` would apply repo A's config to repo B.
- **Single `asyncio.run()` wraps supervise + shutdown** (audit-fix M2).
  Task-to-event-loop binding stays consistent; shutdown errors no
  longer mask the real exception.
- **Strict config type validation** (audit-fix M5). Malformed
  `supervisor.yaml`/`.json` raises a specific `ValueError` instead of
  silently coercing `"yes"` to `True`.
- **`SupervisorPersistence.list_decisions()` public API** (audit-fix
  L1). `_cmd_decisions` no longer pokes `store._connect()`.
- CLI surface: `run "<prompt>" [--cwd DIR] [--session ID] [--config PATH]`,
  `status --session ID`, `decisions --session ID [--limit N]`.
- `SupervisorConfig.from_yaml()` with `_parse_minimal_yaml` — a
  flat-keys-plus-lists subset. `.json` alongside `.yaml` wins when
  present (users who need complex configs pre-materialise to JSON).
- `HaikuEscalator` is a typing `Protocol`; no Anthropic SDK import,
  so the supervisor module stays credential-free. Callers that own an
  API key wire the concrete escalator; v1.2.0 ships with `escalator=
  None` as the default-deny safe default.

## Problems / Solutions

### P1: Supervisor was observational, not enforcing (🔴 CRITICAL)
GPT called it: deny decisions were logged and the worker kept going.
The `claude -p --output-format stream-json` protocol does not expose
a pre-tool-invoke hook, so "pre-block" is not on the table. Fix:
every non-approve verdict triggers `runtime.cancel(task_id)` + force
`CANCELLED`. Red-team semantics now match the Haiku prompt.

### P2: Silence detector was telemetry-only (🟠 HIGH)
`_silence_poller` called `tick()` and threw the return away. A hung
worker would stall the event loop indefinitely. Fix: poller now
cancels the worker when `tick()` returns `POSSIBLY_COMPLETE`.
Detector's grace + median-seed logic still gates the firing
(60 s grace + ≥3 gaps before the adaptive clamp can trip).

### P3: CLI ran two separate event loops (🟡 MEDIUM)
`asyncio.run(supervise)` then `asyncio.run(shutdown)` — tasks bound
to the first loop could misbehave when shutdown touched them on a
different loop. Fix: single wrapper `_run_once()` with
`await runtime.shutdown()` in its `finally`, wrapped in one
`asyncio.run(_run_once())`.

### P4: Default config path ignored --cwd (🟡 MEDIUM)
Running from repo A with `--cwd=B` applied repo A's `.claude/
supervisor.yaml` to a repo B worker. Fix:
`base_dir = Path(args.cwd).resolve() if args.cwd else Path.cwd()`,
then `base_dir / ".claude" / "supervisor.yaml"`.

### P5: Loop-guard counted denies as approvals (🟡 MEDIUM)
3 prior denies of the same `(tool, args)` would trip loop-guard on a
legitimate 4th approve. Fix: filter to `decision == "approve"` rows
only before the `len() >= 3` check.

### P6: Config silently coerced bad types (🟡 MEDIUM)
`bool("yes") → True`, `set("pytest") → {'p','y','t','e','s'}`. Fix:
strict `isinstance` checks on each key with specific `ValueError`
messages that name the file.

### P7: CLI reached into persistence private attr (🟢 LOW)
`store._connect()` from `_cmd_decisions`. Fix: added public
`list_decisions(session_id, limit)` to `SupervisorPersistence`.
