## Summary

Dmitry reported the status "не обновляется": right after a $0.146 Fable task it showed `sess $0.0000 · today $0.0000`. RECON on runtime state (not docs) proved the cache **was** refreshing (`updated_at` seconds old) and the numbers were **arithmetically correct** — the bug was display *semantics*, not data:

- `session` counts only the current CC session; the $0.146 was spent in a **different** project (AINEWS sre-agent, session `9b264f14`) → honest $0 for this Booster session.
- `today` is the Dubai +4h **calendar** day; it was just past Dubai midnight (00:07, 7 Jul) and the last Fable event was 19:46 Dubai 6 Jul → honest $0.
- Net: both legit-zero in a very common case → widget looks broken.

**Fix 1 (commit `c95a419`):** switched both surfaces to `last_task` + `mtd` (always populated once any spend exists). Display-only — the summary schema (session/today/last_task/mtd all still emitted) untouched.

**Locale bug surfaced mid-fix:** `printf '%.2f'` honours `LC_NUMERIC` → on this host it emitted `$0,15` and thousands-grouped `$0,000.1460` (exactly the garbage Dmitry saw next). Replaced printf with **jq** integer-cents formatting from `*_cost_usd_nanos` (jq always uses `.`): cents = `round(nanos/1e7)`, padded to `D.CC`; MTD = whole dollars `round(nanos/1e9)`. Proven under forced `LC_ALL=de_DE.UTF-8` — still `$0.15`.

**Fix 2 (commit `bc70338`):** Dmitry asked to bring `today` back. Restored it as a **third** segment (not a replacement) so a $0 today never reads as broken — `last`/`mtd` stay non-zero anchors while `today` shows the running Dubai-local-day total (displayed even at $0.00, which is the whole point). tmux uses compact single-letter labels (`l`/`d`/`m`) because `status-right-length 40` is shared with pane_title + clock; CC statusline (own row, no cap) uses full words.

## Problems / Solutions

- **"Status not updating" (false lead).** Cache `updated_at` was seconds old — refresh worked. Root cause was semantics: `session`/`today` both legit-$0. → Switched anchor to last+MTD; restored today as third segment.
- **`printf '%.2f'` locale corruption** (`$0,15` / `$0,000.1460`). → Format money in jq from integer nanos; never printf. Proven under de_DE.
- **jq drops the decimal on whole values** (`$1` not `$1.00`) → failed the tmux test regex `\$[0-9]+\.[0-9]+`. → Build cents as integer and pad to `D.CC` in jq.
- **tmux `status-right-length 40`** shared with pane_title+clock → full labels overflow (44–46 chars). → Compact `F l$ d$ m$` (37 chars) on tmux only; full words on the CC row.
- **A "GUARD non-mutation" test FAIL** appeared once → it was my own concurrent manual statusline render firing a background `refresh-session` that touched the real cache mid-suite. Isolated re-run = 20/20. Not a code defect.
