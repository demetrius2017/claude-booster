## Problems / Solutions (major ones)

### P1: Sub-agent was structurally trapped by its own gate

Gate fired in every session uniformly → sub-agent couldn't do the work
→ picked the only available escape (the bypass file). Fix: gate
auto-skips when `agent_id` / `agent_type` set — the gate's purpose
(force Lead to delegate) is already satisfied once we're inside a
delegated session.

### P2: Single-signal brittleness

Original plan used `bool(agent_id)` only. PAL/gpt-5.4 review flagged
that a field rename in a future Claude Code version would reopen the
incident vector. Fix: `is_subagent_context` returns True if **either**
`agent_id` OR `agent_type` is non-empty. Verified empirically with
stdin `{"agent_type":"Explore"}` (no `agent_id`) — gate auto-skips.

### P3: Fail-open on malformed stdin

`json.loads(raw or "{}")` returning `{}` → empty tool_name → allow.
Fix: after JSON parse, require `isinstance(data, dict)`. On fail:
fail-closed with exit 2, neutral stderr, block-logged. Tests for
malformed JSON (`abc{`), JSON array (`[]`), JSON string (`"hi"`).

### P4: ask_gate log was a sensitive-data sink

Every allow and block row stored `message_excerpt: text[:200]` raw.
Assistant messages can contain user-pasted tokens/keys. Fix: drop
excerpt on allow path (matched_pattern is enough for analytics); on
block path, run through `redact_secrets()` (regex matching
api_key/token/secret/password/bearer assignments + JWT-shaped
strings) before the 200-char truncation.

### P5: Telemetry path divergence

`telemetry_agent_health.py` hardcoded `~/.claude/logs`, gates honoured
`CLAUDE_HOME`. Any install or test with `CLAUDE_HOME` set would have
`/start` silently miss bypass attempts. Fix: telemetry now imports
`logs_dir` from `_gate_common` and resolves lazily per call.
