## Institutional lessons (candidates for `institutional.md`)

1. **Round 2 audit is mandatory, not optional.** Round 1 declared the work complete pending one external review. Round 2 found a HIGH + 2 MED that round 1's author + GPT's first pass both missed. The session 3 lesson generalizes: fix commits can introduce new bugs, and narrowed context reveals issues the first pass couldn't see. Always round-trip the re-review.
2. **Simple guards beat elaborate metadata plans.** Phase 2c's full `preserve` column + wiring is correct but non-trivial. A 4-line `raise ValueError` in `consolidate()` closes the same data-loss path today with zero schema change. Ship the guard, plan the full solution.
3. **Dry-run contracts must be actually dry.** A `--dry-run` that silently triggers a schema migration is worse than no `--dry-run` at all, because the user trusts the flag. Audit every transitively-called function in the dry-run path.
4. **Verify the remote exists before promising a "push".** The Claude_Booster repo had no `origin` for months. The workflow assumption that `git push` just works is invalidated on first-time repos; check `git remote -v` before committing to a push-deploy.
