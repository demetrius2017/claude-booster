## Problems / Solutions

| Problem | Solution |
|---------|----------|
| compact_advisor.py emitting ~200 log lines/hour for `marker_exists`, `below_threshold`, `no_marker` | Patched to return early (no log write) for those three paths; updated acceptance test to assert silence on 10 below-threshold calls in a row. |
| `/simplify` round 1 — compact_advisor double-stat: `marker_path.exists()` then `marker_path.stat().st_size` = two syscalls with TOCTOU window | Collapsed to single `try: size = marker_path.stat().st_size` / `except FileNotFoundError: return`. |
| codex `~/.codex/state_*.sqlite` schema not documented anywhere | Explore agent discovered it via `sqlite3 .schema` on an actual DB file; `tool_calls` table has `input_tokens + output_tokens` columns per call timestamp. |
| `_get_codex_quota_pct()` needed to handle missing DB gracefully (Codex not installed / no usage yet) | Returns `0.0` (not an error) when no `state_*.sqlite` files found — balancer treats Codex as zero-pressure in that case. |
| `_score_candidate` previously took `weekly_max_pct: float` as a positional arg — brittle when adding second budget dimension | Refactored to `budget_pcts: dict[str, float]` keyed by provider string; both `anthropic` and `codex` paths work; old callers updated. |
| sort key crash in `_get_codex_quota_pct`: `p.stem.split("_", 1)[1]` on a file named `state.sqlite` (no underscore) raised `IndexError` | Fixed to `p.stem.split("_", 1)[-1]` which degrades gracefully to the full stem; `import re` also added (was used but not imported). |
| Nested `try/except` in `_get_codex_quota_pct` was catching `Exception` broadly, masking real errors | Narrowed to `sqlite3.OperationalError` + `FileNotFoundError`; outer `except Exception` replaced with explicit DB-connection guard. |
| PATH override in `settings.json` (env: `{PATH: "...:/Users/.../.claude/scripts"}`) replaced entire system PATH — would break `git`, `python3`, `node` on next session start | Removed the override; added `export PATH="$HOME/.claude/scripts:$PATH"` to `~/.zshrc` instead; tool-strategy.md recipe updated to use full absolute path `~/.claude/scripts/codex_worker.sh`. |
| gpt-5.3-codex-spark intelligence score = 15 (UNKNOWN fallback) — model was absent from `openai_models.json` | Added entry with `intelligence: 17`; now scores correctly in Pareto. |
| read-only DB access in `/simplify` round 2: `_get_codex_quota_pct` opened DB with `sqlite3.connect(path)` (read-write mode), which could create WAL files or cause contention | Changed to `sqlite3.connect(f"file:{path}?mode=ro", uri=True)` — OS-level read-only, no WAL, safe from parallel writers. |
