## Проблемы / решения
- **zsh nomatch + parallel cascade в /start.** Корневой анализ: zsh выкидывает ошибку на этапе парсинга (до redirects), harness отменяет всех siblings при exit≠0 одного параллельного call'а. Воспроизведено на сессии-старте: 6-параллельный recon → 4 отменённых telemetry-вызова из-за одного `ls roadmap.* 2>/dev/null`. Фикс — `Read` для file-existence-проб, не Bash-globs; для unavoidable globs на zsh — `(N)` qualifier; никогда не группировать fragile probes с критическим telemetry в одном parallel block. Закодифицировано в `core.md` + `commands.md` (templates + ~/.claude/rules sync).
- **PAL gpt-5.5 cache miss.** Не баг — known behavior: PAL MCP читает реестр моделей один раз при старте процесса. Verify требовал свежей сессии (т.е. этой). Закрыто.
- **Figma Starter rate-limit gate (6 calls/mo).** Не баг продукта, а business-decision pending. Ставит пилот в режим "тщательно тратить вызовы", не "итерировать".
- **Push отклонён harness'ом.** При попытке `git commit && git push origin main` единым chain'ом hook отказал ("Pushing directly to main bypasses pull request review"). Workaround: split commit (allowed) + push (требовал явного user-approval). Пользователь явно дал доступ — push прошёл `68afd50..90f1c01`.
