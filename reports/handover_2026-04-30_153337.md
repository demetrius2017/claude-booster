## PAL hardening details

GPT-5.5 thinkdeep, continuation `27613123-a244-49a7-95ba-baeaab0dbf9a`. Применённые правки:

1. **Verifier knowledge boundary** — явные списки: видит публичный контракт (interface, signatures, paths, schema, endpoint), не видит Worker's prompt / plan / reasoning / drafts / Lead-комментарии о выбранной стратегии.
2. **Failure classification protocol** (W/V/A/E):
   - W. Artifact wrong → новый Worker, тест не менять
   - V. Test invalid / over-constrained → новый Verifier с regen теста, Worker не менять, Lead не override'ит вердикт
   - A. Contract ambiguous → Lead уточняет Artifact Contract, перезапуск пары
   - E. Environment → чинит harness, тот же тест
3. **Test Legitimacy Standard** — детерминизм, наблюдаемое поведение, диагностический stdout, без localhost/network/timing/CWD-зависимостей, tempdir + cleanup.
4. **Skip carve-out сужен отрицательно** — список impact-классов где skip запрещён (behavior change, bug fix, auth, migration, concurrency, test-infra, prod config, multi-file где взаимодействие важно, after prior failures, audit-driven). «Small» / «obvious» сами по себе не достаточны.
5. **Artifact Contract** обязательное предусловие — Objective / Verified Facts Brief / Artifact path / Invocation / Inputs / Expected observable behavior / Out of scope / Environment constraints / Acceptance emphasis. Без него пара не спавнится (Adzic «definition of ready»).
