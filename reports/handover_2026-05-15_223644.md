## Summary

Consilium (4 agents: Tooling Architect, Budget Engineer, Product Strategist, Infrastructure Engineer) investigated why Codex CLI had 0% utilization and PAL MCP crashed at startup. Three root causes found and fixed:

### 1. PAL MCP crash — `provider_type` field in openai_models.json
- Two Codex models had custom `provider_type: "codex-cli"` field from model_balancer development
- PAL's strict schema validation rejected unknown fields → 0 models loaded → crash
- **Fix:** Removed `provider_type` from `~/.claude/openai_models.json` (2 models)
- PAL doesn't need this field — model_balancer.json already tracks provider routing

### 2. Enforcer Phase 2 deadlock — tag check blocked everything
- CC bug #16598 prevents `updatedInput` auto-inject of `[model]` tag
- Phase 2 hard-blocked (exit 2) every Agent call without tag → all calls blocked
- **Fix:** Phase 2 demoted to advisory (exit 0 + stderr warning)
- Architectural insight from Tooling Architect: "PreToolUse hooks fire AFTER tool selection — they can validate but cannot redirect"

### 3. Enforcer Phase 1 — tier mismatch blocked stronger overrides
- high_blast_radius routes to sonnet, but consilium agents legitimately request opus
- Phase 1 blocked opus→sonnet without checking that opus > sonnet
- **Fix:** Added tier rank comparison — stronger model always passes

### Strategic shift (Dmitry's directive, post-consilium)
- **PAL → Codex:** `codex_worker.sh gpt-5.5` replaces PAL as primary external review (flat-fee vs API tokens). PAL demoted to optional fallback.
- **Bio-agent tier:** New routing category for consilium/audit/hackathon agents → Codex gpt-5.5 via Bash, not Agent tool.
- **Escalation rule:** Never retry at same model tier on failure. Haiku→Sonnet→Opus, codex-5.3→codex-5.5→Opus.
- **Sonnet narrowed:** Use only for pure mechanical work (grep-replace, formatting, boilerplate). Reasoning tasks escalate.
