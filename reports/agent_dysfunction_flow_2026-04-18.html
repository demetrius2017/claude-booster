Три-частный continuity failure

Consensus 7 агентов (5 Claude panel + 2 GPT). Любой один механизм недостаточен — ломается связка.

1

Authority без supersession

institutional.md и per-project memory хранят March findings как canonical knowledge с формулировкой "Permanent knowledge — never auto-prune". April-17 аудит "structurally wrong" лежит в reports/ без link back в rules.
stale canon Δ +23 days
no [SUPERSEDED by] metadata
"bullying" effect — rules > reports
2

Rules as prose, not mechanics

pipeline.md требует "VERIFY → curl API on prod". commands.md требует RECON before opinions. Оба правила загружены в контекст. Handover'ы их не выполняют. Missing forcing function = hook который блокирует wrong action, а не ещё одно правило описывающее right action.

3

Runtime reintroduces divergence

Система сама создаёт fresh divergence на каждом deploy быстрее чем сессия её чинит. Async startup reconcile + dead broker-snapshot write path (total_value <= 0 skip) + is_reconciliation NAV exclusion. Агент лечит симптом, ставит PASS, deploy снова.

Ирония. Правила pipeline.md и commands.md уже содержат всё необходимое: VERIFY с curl после deploy, RECON before opinions, "not say completed without evidence". Они загружены в каждую сессию. Но они — проза, а не blocking mechanics. Агент читает, кивает, пропускает. pipeline.md:13 — "VERIFY · Real commands/curl/scripts. After deploy — curl API on prod. Collect EVIDENCE."
