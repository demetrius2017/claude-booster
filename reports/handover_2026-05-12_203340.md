## Summary

The session opened with a clear economic pressure: Claude Max weekly limit was at 86% on day 5 of the prior session, with root cause traced to 38% of general-purpose subagents running on Opus by default. v1.7.0 had already gated large rule files and injected `model: "sonnet"` directives where mechanical. v1.8.0 takes the next logical step — instead of static routing tables in tool-strategy.md, give the Lead a live, self-updating daily decision file (`~/.claude/model_balancer.json`) driven by actual per-category performance data accumulated in rolling_memory.db. The design has two phases: Day-1 passive (seed reasonable defaults and start collecting data) and Day-N active (Pareto-score on accumulated metrics and override the seed). The Pareto formula is `score = 0.5*intelligence - 0.3*norm_lat*20 - 0.2*budget*20`, where `norm_lat` and `budget` are normalized 0–1 across candidates. Two categories are permanently pinned and never subject to active override: `lead` (always claude-opus-4-7) and `high_blast_radius` (always claude-sonnet-4-6 with `applies_to=[auth, security, secrets, db_migrations, financial_dml, infra_config]`).

Day-1 was built as 7 sequential steps with paired Worker+Verifier at each: codex_worker.sh + RECON/consilium groundwork (step 1), model_metrics DDL and schema v7 migration (step 2), model_metric_capture PostToolUse hook (step 3), daily decision engine model_balancer.py (step 4), SessionStart injection of routing/limits summaries (steps 5a–5c), settings.json hook registration (step 5c), and a unified smoke wrapper `test_model_balancer_all.sh` (step 7). Day-N active decide() was then layered on top as a separate feature — it reads the last 14 days of model_metrics, groups by (provider, model, task_category), Pareto-scores the candidates, and writes the winning choice into model_balancer.json. The bypass env `CLAUDE_BALANCER_DISABLE_ACTIVE=1` preserves Day-1 passive behavior; `CLAUDE_BALANCER_FORCE_ACTIVE=1` ignores the MIN_SAMPLES guard. The session closed with a /simplify pass: 3 lens agents (reuse, quality, efficiency) surfaced 14 findings, 11 were fixed (including stringly-typed provider constants, timestamp format drift causing silent query mismatch, and a TOCTOU `.exists()→.read_text()` race), 3 were deliberately skipped with rationale. README updated to v1.8.0, install validated (dry-run + --yes + post-install smoke), and all 13 commits pushed to both `origin` (private) and `public` remotes.
