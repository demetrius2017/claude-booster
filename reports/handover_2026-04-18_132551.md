### C6 — verify_gate.py (~3h, ✅)

**File:** `~/.claude/scripts/verify_gate.py` (v1, 490 LOC).

**Activation (narrow v1 scope):**
- Fires ONLY when `tool_name='Bash'` AND command matches `\bgit\s+(?:commit|add)\b` AND references `reports/handover_YYYY-MM-DD*.md`.
- TaskUpdate(status=completed) gating DEFERRED to v2 — would be too noisy at every task.

**Three-tier mode precedence:**
1. `VERIFY_GATE_MODE` env var (test override)
2. `.claude/CLAUDE.md` YAML frontmatter `verify_gate: enforcing|warn|off`
3. Auto-detect: `warn` if `deploy/` or `.github/workflows/` exists, else `off`

**Validation rules:**
- `status='pass'`: ALL evidence entries must match a marker (curl/wget/psql/sqlite3/SELECT/PRAGMA/HTTP/docker/kubectl/DevTools/pytest/exit=N) AND NOT match any fake-pattern (localhost/127.0.0.1/0.0.0.0/::1, `|| true`, `curl -s` without --fail/-o/| tee/-w/-S/> redirect) AND include HTTP status (1xx-5xx) for HTTP markers / rowcount for SQL markers.
- `status='na'`: `reason_na` non-empty AND `git diff --cached --name-only` entirely in allowlist (`docs/`, `doc/`, `reports/`, `audits/`, `.claude/`, `tests/`, `test/`, suffixes `.md/.txt/.rst`, basenames README*/LICENSE/CHANGELOG/.gitignore).

**Decision output:**
- allow → exit 0, no stdout.
- enforcing + block → exit 2 + JSON `{"hookSpecificOutput": {"hookEventName": "PreToolUse", "permissionDecision": "deny", "permissionDecisionReason": "..."}}`.
- warn + block → exit 0 + `[verify_gate] WARN: ...` stderr.

**Logging:**
- `~/.claude/logs/verify_gate.log` — WARN/ERROR (RotatingFileHandler, 500KB × 3).
- `~/.claude/logs/verify_gate_decisions.jsonl` — append-only, every fire, weekly distribution review.

## Known v1 limitation + v2 design brief (HIGH #3 deferred)

**Problem:** Hook validates evidence text in isolation — pattern matching on strings the agent supplied. An agent could emit a plausible-looking curl/SQL output without actually running the command, or replay output from before the change.

**Why deferred (not done in v1):**
- Tying evidence to actual `tool_use`/`tool_result` transcript records requires heuristic mapping (evidence text → specific tool call's stdout/stderr). False-positive rate could frustrate legitimate sessions.
- Each tool outputs in different shapes (Bash stdout, DevTools structured response, sqlite3 table). A unified matcher is non-trivial.
- The other 4 HIGH/MEDIUM fixes already remove the bulk of fake-evidence surface (localhost/loopback/`|| true`/curl -s, no HTTP status, missing rowcount, stale block).

**v2 design brief** (target 2026-05-15 — 1 month):

1. Extend `_extract_assistant_text` to also return tool_use / tool_result records with their ids.
2. For each evidence entry, require at least one distinctive substring from the entry (URL host + path, SQL rowcount number, specific status code) to appear in some `tool_result` body text from the last 200 lines.
3. If no `tool_result` blocks exist in the window (all-reasoning session), fail open to avoid false-positive blocks. Log the bypass to telemetry.
4. Add KPI: "Evidence-backed vs plausible-only" ratio surfaced by `telemetry_agent_health.py`.
