## Problems / Solutions

| Problem | Root cause | Solution / evidence |
|---|---|---|
| Ten-line Lead character prompt could fade into long context or become wallpaper if repeated before every tool. | Abstract identity prompts are priming, not enforceable behavior; indiscriminate repetition causes habituation and context growth. | Distributed concise reminders across SessionStart, phase transitions, compaction recovery, delegation, synthesis, and verdict boundaries; retained machine gates for mechanically checkable requirements. |
| Binary or malformed `.claude/.phase` could crash reminder injection. | Hook assumed a valid text phase value. | Added defensive parsing and fallback behavior; Lead-anchor suite passes with 0 failures. |

| Telemetry records could interleave under concurrent append and grow without bound. | Append path lacked serialization and record-size enforcement. | Added bounded, concurrency-safe emission and coverage for large/concurrent records; memory hardening suite passes 27/27. |
| Static `claude_max_weekly_used_pct=0.86` conflicted with live quota state. | A frozen seed had become a competing source of truth. | Removed the seed; routing now trusts only fresh live-cache data, with conservative behavior when stale/missing; rate-limit suite passes 15/15. |
