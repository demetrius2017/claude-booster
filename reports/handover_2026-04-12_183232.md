# Handover: 2026-04-12

## Summary

Реализован Karpathy's compounding knowledge pattern поверх существующей rolling memory системы. Раньше `session_end.py` извлекал факты и просто INSERT'ил их — новое знание не интегрировалось с существующим. Теперь: умный merge при ingestion (без LLM, в hook) + periodic consolidation через Claude Haiku.

## What was done

### 1. Level 1 — Smart ingestion в `~/.claude/scripts/rolling_memory.py`

Новые функции (все без LLM, выполняются в hook path):
- `_extract_keywords(text)` — whitelist regex `[a-zA-Z0-9_./:-]{3,}` + stopwords filter
- `_similarity_metrics(a, b)` — возвращает `(subset_score, jaccard, shared_count)` для строгого gate
- `_fts_quote(term)` — безопасное quoting всех терминов для FTS5 MATCH
- `_find_similar(content, memory_type, scope)` — FTS5 поиск с scope filtering (same scope OR global, приоритет same-scope через ORDER BY CASE)
- `_merge_content(existing, new)` — добавляет только novel части, capped at `MERGE_MAX_LENGTH=2000`
- `memorize_with_merge(...)` — decision tree: MERGE / LINK / NEW с atomic CAS update (compare-and-swap на content_hash с retry)

Merge gate (строгий — требует все три):
```python
MERGE_SUBSET_THRESHOLD = 0.6   # |A∩B| / min(|A|, |B|)
MERGE_JACCARD_THRESHOLD = 0.4  # |A∩B| / |A∪B|
MERGE_MIN_SHARED = 3           # абсолютный floor, чтобы не мёржить на 1-2 generic токенах
MERGE_LINK_THRESHOLD = 0.35    # partial overlap → LINK via metadata.related_to
```

### 2. Level 2 — Periodic consolidation (с Claude Haiku)

- `_cluster_memories(memories, threshold=0.3)` — BFS по similarity graph, транзитивные кластеры (A~B + B~C → один кластер)
- `consolidate(scope, memory_type, dry_run)` — группирует similar memories, синтезирует каждый кластер через Haiku, **atomic**: INSERT synthesized → deactivate originals в одной транзакции. При failure — rollback, оригиналы остаются active
- `_get_anthropic_client()` — API key из env или `~/.claude/.credentials.json`
- Поддержка CONFLICT маркеров: prompt просит Haiku пометить противоречащие факты
- Priority bonus: synthesized entry получает `max(cluster.priority) + 5`
- Metadata tracking: `consolidated_from: [<old_ids>]` для провенанса

## Audit Findings (PAL MCP → GPT-5.4)

Code review нашёл **3 HIGH** и **3 MEDIUM** issues — все исправлены перед deploy.

| # | Severity | Issue | Fix |
|---|----------|-------|-----|
| 1 | HIGH | Merge gate слишком permissive (`intersection / min`) — короткая generic запись могла дать score 1.0 и смёржить distinct lessons | Требование всех трёх: `subset >= 0.6 AND jaccard >= 0.4 AND shared >= 3` |
| 2 | HIGH | Cross-scope contamination — `_find_similar` только бустил same-scope на 15%, не фильтровал. Project-специфичные ошибки могли мёржиться между разными repos | `WHERE scope = ? OR scope = 'global'` с ORDER BY CASE для приоритета |
| 3 | HIGH | Non-atomic consolidation — `forget()` оригиналов перед `memorize()` synthesized. При failure = data loss (кластер исчезает из active) | `BEGIN IMMEDIATE` → INSERT synthesized → deactivate originals → COMMIT. Failure → rollback, оригиналы остаются |
| 4 | MED | Race condition в merge UPDATE — два параллельных hook могли overwrite друг друга | Compare-and-swap на `content_hash` с retry (2 попытки) |
| 5 | MED | FTS5 quoting только для non-alnum токенов | Quote ALL terms через `_fts_quote()` с escaping `"` → `""` |
| 6 | MED | `_merge_content` мог расти неограниченно | Cap `MERGE_MAX_LENGTH=2000` с маркером `[...truncated, needs consolidation...]` |
| 7 | LOW | Non-transitive clustering (greedy single-pass) | BFS over similarity graph |

## Problems & Solutions

| Problem | Solution |
|---------|----------|
| Merge gate через `min(|a|,|b|)` давал false positives | Добавлен AND gate: subset + jaccard + shared_count |
| Cross-repo error contamination | Scope filtering в WHERE + ORDER BY CASE |
| Consolidation terять данные при API failure | Atomic INSERT-then-deactivate с rollback |
| FTS5 query errors на токенах с дефисами (`cap-drop`) | `_fts_quote()` для всех терминов |
| `_merge_content` рос бесконечно | Cap 2000 chars с маркером truncation |
| Non-transitive clusters разрывали A~B~C | BFS по similarity graph |
