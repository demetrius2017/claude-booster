## Problems / Solutions

Problem: The desired concept was not a generic "lesson" or build-time warning. It needed a name and a memory boundary for failures that happened after deploy in real project usage.

Solution: Named it **Incident Register** and implemented it as a separate `incident` memory type with priority `95`, forced preservation, optional severity metadata, and dedicated rendering headers (`=== INCIDENT REGISTER ===` / `=== INCIDENT WARNINGS ===`).

Problem: Existing report indexing was consilium/audit-oriented and could accidentally accept malformed or conflicting frontmatter.

Solution: `index_reports.py` now infers `incident` from filename/frontmatter, rejects non-string `type`, skips filename/frontmatter conflicts, resolves paths for idempotency, and keeps incidents out of `error_lesson`.

Problem: `/start` and `/go` could still ignore the new lane if it was only indexed.

Solution: Updated both command templates. `/start` must read listed incident sources before planning. `/go` must load incidents immediately after Artifact Contract validation and inject a compact `Incident Warnings` block into the Flow Designer prompt.

Problem: Fresh incident reports would not become visible until a manual reindex.

Solution: `memory_post_tool.py` now triggers `index_reports.py` for `reports/incident_*.md` writes, same as consilium/audit reports.
