## Autonomy layers (for when things go wrong)

| Failure mode | Supervisor response |
|---|---|
| Worker hits CLI internal turn limit (`error_max_turns`) | Auto-respawn with `--resume <cli_session_id>` + "continue the task" prompt, up to `max_continuations` (default 5) |
| CLI cuts off mid-tool-use (`aborted_streaming`) | Same auto-respawn path — gate now retries on ANY non-success subtype |
| Worker tries to do destructive action (deny-list hit) | Cancel via SIGINT→SIGTERM→SIGKILL ladder; decision persisted as `deny` |
| Worker hangs (silence > adaptive threshold) | Poller forces `CANCELLED`, kills worker |
| Quota budget exhausted (85% usage) | Circuit `OPEN`, future `admit()` calls refused |
| Main Claude fallbacks to inline work | `delegate_gate` blocks 2nd+ direct Bash/Edit |
| Main Claude asks "Apply patch?" | `ask_gate` Stop hook refuses, forces continuation |
| Main Claude asks "A or B?" | `ask_gate` multi-option regex catches + refuses |

## Problems / Solutions (major ones)

### P1: Field logs showed Claude ignoring soft pipeline.md
Dmitry: "не нужно правил должно быть гейты вместо правил — правил Claude игнорит"
→ Shipped `delegate_gate.py` (PreToolUse) and `ask_gate.py` (Stop) as physical hooks.

### P2: Worker died at max_turns, user had to manually split tasks
→ `8df456d`: auto-continuation via `--resume <cli_session_id>` + "continue the task" prompt.
→ `41efa53`: broadened gate to ANY non-success subtype (not just `error_max_turns`), because real CLI sometimes returns `aborted_streaming`.

### P3: Worker still asked "A or B?" in field logs
Root cause: worker got vanilla Claude Code behaviour.
→ `aaeaecb`: inject `AUTONOMY_DIRECTIVE` as `--append-system-prompt` + `--permission-mode auto`.

### P4: Main Claude kept asking "Apply patch?" after research agent returned
Root cause: `core.md` 51% rule carved out "production deploys require confirmation".
→ `f5fd1b3`: narrowed carve-out to genuinely destructive/irreversible (rm -rf, force-push, DB schema, external messages). Code deploys are PRE-approved by task-giving.
→ `e3f0cb8`: added `ask_gate.py` as the hard enforcement since the rule edit is still soft.

### P5: `/supervise` UX — too many magic words
Field: `/supervise run "quoted task"` friction.
→ `5540da9`: `/supervise <free-text>` auto-promotes to `run`, quotes optional.
→ `c4a9bd5`: bare `/supervise` prints help + sessions table, not argparse error.

### P6: No way to stay up-to-date
→ `b4183d5`: `check_booster_update.py` SessionStart hook. Warns on drift; `CLAUDE_BOOSTER_AUTO_UPDATE=1` auto-runs install.py.

### P7: Multiple audit-caught CRITICALs in Session 3/4
GPT-5.4 via PAL caught: multi-block assistant message drop, observer-vs-enforcer
semantic mismatch (deny decisions logged but worker kept running), silence-poller
return ignored, settings.json drift not re-merged on BOOSTER_SAME. All closed with
regression tests.
