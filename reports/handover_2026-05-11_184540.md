## Summary

The session started from a concrete user concern: Claude Max weekly limit was at 86% on day 5. Anthropic stats told the story clearly — 98% subagent-heavy sessions, 38% general-purpose subagents running at Opus by default, >150k context in 43% of turns, and /handover accounting for 5% of weekly skill usage. Root cause analysis identified two primary levers: three large rule files loading unconditionally on every /start (33,766 tok always-on per telemetry), and command files that don't enforce explicit `model:` in agent spawns, leaving Lead to default to Opus under context pressure.

Token-budget reduction proceeded on two fronts. First, paired-verification.md (29kB) and quality-no-defects.md (18kB) were gated behind description+paths frontmatter so they only load on coding-scope sessions. Explicit `model: "sonnet"` directives were injected into /handover to stop that skill from silently burning Opus on mechanical work. An attempt to apply the same treatment to /consilium and /lead was vetoed by the user — those commands are deliberately invoked for hard problems where Opus is the right tool and the cost is priced in; the veto was saved as feedback memory so this won't be re-litigated. Second, a deterministic alternative to "Lead self-discipline at 120k" was built: two new hooks — compact_advisor.py (PostToolUse) writes a one-shot marker file when the token threshold is crossed; compact_advisor_inject.py (UserPromptSubmit) reads the marker and injects an advisory into the next prompt via additionalContext, then deletes the marker. Bypass available via CLAUDE_BOOSTER_SKIP_COMPACT_ADVISOR=1. All events logged to ~/.claude/logs/compact_advisor.jsonl.

The session then dogfooded its own /audit pipeline on the new feature. Three Sonnet lens agents (correctness, security, operational) found 2 HIGH (S1: path traversal via session_id — arbitrary marker writes; O1: module-level int() crash on malformed CLAUDE_BOOSTER_COMPACT_THRESHOLD env value) plus 3 MED and 6 LOW findings. All HIGH and all MED findings were remediated in-session via paired Worker+Verifier. The /simplify pass on top of that surfaced a convergent finding across all three lens agents: the `_log_event` helper was duplicated identically between both new hook scripts (24 LOC), bypassing the existing `_gate_common.append_jsonl` shared utility already used by delegate_gate and ask_gate. Refactored to use the shared utility — net -16 LOC, and all 66/66 acceptance assertions remained PASS. Full audit report is at `reports/audit_2026-05-11_compact_advisor.md`. VERSION bumped 1.5.0 → 1.7.0, README appended with "What's new in v1.7.0", 7 commits pushed to both `origin` (private Claude_Booster) and `public` (claude-booster) remotes.
