# Handover — Autopilot CADENCE class + always-on proceed-to-boundary rule

## Summary

Two shipped, pushed commits on clean `main` fix Dmitry's complaint that Fable
autopilot "still stops with dumb questions":

- `a5249c8` — adds a **CADENCE** classification to the autopilot hook
  (`templates/scripts/fable_autopilot.py`). Pure timing/sequencing questions
  ("start now or next session?", "commit now or later?") under active autopilot
  are intercepted with a static "proceed on the roadmap default" instruction —
  never routed to Dmitry, never consuming a Fable consult slot. `_requires_user`
  (hard-authority boundary) is untouched and stays first in the chain.
- `7088bcc` — promotes the **behavioral half** to an always-on rule
  (`templates/rules/autopilot-behavior.md`), because the regex hook cannot catch
  the recurring form: a bundled next-step MENU ("layer 2 UI / push / backup /
  handover — or continue?") that mixes a real boundary (UI acceptance, push)
  with delegatable work.

Root-cause insight: the fix is two-layered by necessity. The **hook** kills pure
cadence (regex). The **behavioral rule** kills the bundled menu (only Lead
behavior can separate "the UI part is yours" from "the coding part I just do").
The v1 fix mistakenly put the behavioral rule in the `/autopilot` command doc —
not always-on — so a long-running Lead never carried it; v2 (`7088bcc`) moved it
to `~/.claude/rules/`. Confirmed live in Dmitry's parallel electro-estimate-ai
session: after restart the Lead self-corrected ("остановился на включении — моя
ошибка, продолжаю по roadmap") and proceeded autonomously to A3 layer 2, holding
the visual-acceptance boundary.

Built via `/go fable` (Семёрка-F): Flow Designer (gpt-5.6-sol) → Fable Challenge
(ADDITIVE_GAPS, 5 watchlist items, 4 contract ambiguities resolved by Lead) →
read-only Prototype Gate (proved the Stop leak empirically) → Worker
