### 3. supervisor/ module skeleton (task #6) — 508 LOC code + schema
`templates/scripts/supervisor/` directory, 5 files per consilium §8:

- **`policy.py` (266 LOC)** — Tier 0/1/2 engine.
  - 13 `DENY_BASH_PATTERNS` regexes mirroring
    `settings.json.template` permissions.deny lines 49-66. Parity
    test `test_every_settings_deny_matches_a_policy_pattern` asserts
    no settings entry lacks a policy match.
  - 11 `DENY_PATH_SUBSTRINGS` (`.env`, `id_rsa`, `credentials`,
    `/.aws/`, `/.ssh/`, `/.git/config`, `.npmrc`, `.pypirc`,
    `.netrc`, …) hard-deny Read/Grep/Glob regardless of project.
  - `GIT_SCRUB_FLAGS`: `core.pager=cat`, `core.fsmonitor=`,
    `diff.external=`, `core.sshCommand=`, `protocol.version=2` —
    wraps every Tier 0 git subcommand. Neutralises hostile
    `.git/config` RCE (red-team RT1).
  - `CURL_HARDEN_FLAGS`: `--no-netrc`, `--max-redirs 0`, `--fail`,
    `-sS`, empty `Cookie:`/`Authorization:`. Refuses `curl -X POST`,
    `-d`, `--data-binary`, shell-expansion in URL.
  - Tier 1 (`pytest`, `npm test`, `cargo test`) — default escalate,
    approve only if tool in `ctx.tier1_enabled`.
  - Tier 2 (`npm install`, `pip install`, `cargo build`) — default
    escalate, approve only if `ctx.tier2_trusted_repo=True`.
  - `args_digest(tool, args)` — stable 16-hex SHA256 for §5/Q4 loop
    detection (same (tool,args) approved ≥3× in 5min → escalate).

- **`quota.py` (128 LOC)** — QuotaTracker.
  - `CircuitState`: CLOSED → HALF_OPEN (≥50 %% usage) → OPEN
    (≥85 %% usage).
  - `admit(estimated_worker_tokens) → (bool, reason)` — refuses if
    estimate > worker-budget-remaining, refuses entirely in OPEN,
    degrades to half-budget in HALF_OPEN.
  - Defaults: 15 %% supervisor reserve, 50 000 tok session cap,
    5 h rolling window.
  - `snapshot()` returns a dict shaped 1:1 to the `supervisor_quota`
    SQLite row defined in `schema.sql` — no conversion layer needed.

- **`runtime.py` (91 LOC)** — `WorkerRuntime` Protocol.
  - 7 methods: `submit_task`, `events` (async-iter), `terminal_state`,
    `tool_invocations`, `usage`, `cancel`, `shutdown`.
  - `WorkerEvent` dataclass (kind, task_id, timestamp, payload) —
    transport-independent; `policy.py`/`quota.py`/future `detector.py`
    never import stream-json or MCP specifics.
  - MCP migration triggers documented inline per consilium §5/Q1.

- **`schema.sql` (40 LOC)** — `supervisor_decisions` +
  `supervisor_quota` tables, CHECK constraints on `decision ∈
  (approve,escalate,deny)` and `circuit_state ∈ (closed,half_open,
  open)`. `CREATE IF NOT EXISTS` — safe to apply against live
  `~/.claude/rolling_memory.db` without version bump. Two indexes
  on `supervisor_decisions`: `(session_id, ts)` and `(args_digest, ts)`
  — the latter backs §5/Q4 loop detection query.

- **`__init__.py` (23 LOC)** — module header pointing at consilium
  report + listing what's skeleton-only vs shipping.

### P3: `parents[3]` vs `parents[2]` in test path bootstrap
`templates/scripts/supervisor/tests/test_*.py` → `parents[3]` is
`templates/` (needed scripts), `parents[2]` is `templates/scripts/`
(correct for `from supervisor import policy`). Caught on first run
— `ImportError: No module named 'supervisor'`.

### P4: Deny-parity test probe generator
Replacing `*` with `x` turns `git push -f*` → `git push -fx`, which
the regex `-f\b` (word boundary required) does not match. Switched
to `" x"` (space + placeholder) → `git push -f x` matches. Real
usage (`git push -f origin main`) also matches because space is a
word boundary.
