- Signal 1 (source file recency, 0.7 weight): `git log --follow` → дни с последнего изменения
- Signal 2 (manifest churn, 0.3 weight): `git log -S` → частота мутаций за 90 дней
- 2x2 matrix: STABLE / CODE_DRIFT / INTERFACE_FLUX / DEV
- Cold-start fallback (source_history) когда манифест < 5 коммитов
- Ref-based cache, shallow-clone guard, ThreadPoolExecutor

**3. Consilium** (4 агента + GPT-5.5) — ключевые решения:

- Derived view > stored state (DevOps: chicken-and-egg)

- Manifest absence != safety (GPT-5.5: unknown = unknown risk)

## Problems / Solutions

| Problem | Solution |
|---------|----------|
| Worker agents bypass dep_guard entirely — no architecture awareness during edit | Architecture constraints distributed through Artifact Contract, not hook. Verifier tests downstream consumers. |
| dep_manifest.json can be stale — audit treats stale map as truth | Code-over-docs principle: code = ground truth, manifest = navigation aid. Finding type: architecture-docs-stale |
| How to know which manifest entries to trust? | Dual-signal stability metric: source file recency + manifest churn → 2x2 matrix |
| Commit count conflates velocity with time (30/day team vs 2/week team) | Calendar days as primary axis (Data Scientist consilium recommendation) |
| Stored stability in manifest creates chicken-and-egg (mutation resets counter) | Derived view: ephemeral .cache/arch_stability.json, ref-based invalidation |
| arch_stability.py sys.exit(0) on errors silences failures in pipelines | /simplify caught this → fixed to sys.exit(1) on all error paths |
| Worker used different field names than Artifact Contract (classification vs stability) | W-failure classified, fields renamed to match contract |
| Verifier test sed section extraction captured only heading, not content | V-failure classified — test's heredoc stdin handling bug (python3 - <<EOF consumes stdin for script, sys.stdin.read() returns empty) |
