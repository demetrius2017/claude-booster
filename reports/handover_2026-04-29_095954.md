# Handover 2026-04-29 095954 — Codex bridge

## Summary

Дмитрий локально установил OpenAI Codex CLI (0.125.0) и попросил перенести
паттерны Claude Booster: память, правила, гейты, team works. Сделан минимально
жизнеспособный мост, акцент на **один источник памяти**.

**Что переехало (3 артефакта):**

1. **`~/.claude/scripts/rolling_memory_mcp.py`** (369 LOC, stdlib-only) — stdio
   MCP-сервер по протоколу 2024-11-05. Обёртка над `rolling_memory.py --json`.
   Экспонирует 5 tool'ов: `memory_stats`, `memory_start_context`,
   `memory_search`, `memory_list`, `memory_memorize`. Three Nos guards
   физически в коде: input whitelist + JSON-RPC -32601 на unknown tool, body
   через `subprocess.run(timeout=30)` + try/except TimeoutExpired/FileNotFound,
   output через `json.loads` + проверку top-level `cmd` ключа против ожидаемого
   subcommand'а. Audit log в `~/.claude/logs/rolling_memory_mcp.jsonl`
   (`{ts, tool, returncode, duration_ms}` per call).

2. **`~/.codex/AGENTS.md`** (629 строк, ~57 KB) — конкатенация 8 ядерных правил
   из `~/.claude/rules/`: core, pipeline, tool-strategy, commands,
   communication-style, quality-no-defects, error-taxonomy, deploy. Skipped
   path-gated `institutional_*.md` (узкие per-domain), `frontend-debug.md`
   (path-gated), `_canary.md` (Claude-specific telemetry). Добавлен Codex-
   specific footer о компенсации отсутствующих хуков (verify_gate / phase_gate
   / require_evidence soft-enforced как инструкции).

3. **`~/.codex/config.toml`** — добавлен `[mcp_servers.rolling_memory]` блок с
   абсолютным путём к python3 и серверу. (После моей записи Дмитрий вручную
   добавил `model = "gpt-5.5"` + `model_reasoning_effort = "xhigh"` +
   `approval_mode = "approve"` для трёх tool'ов — это его осознанная настройка,
   не трогал.)

**Один источник памяти:** `~/.claude/rolling_memory.db` (3.7 MB, 161 active row).
И Claude Code, и Codex CLI пишут/читают одну SQLite. Что запомнил один —
видит другой. Подтверждено round-trip'ом: `memorize` через MCP записал
id=17748, `search` его нашёл.

**Что сознательно не переносится (предупредил upfront):**

- Hooks-based gates (verify_gate, phase_gate, require_evidence,
  require_task, preserve_plan_context) — у Codex нет hook surface. В AGENTS.md
  они зафиксированы как soft-инструкции; реального enforcement нет.
- Slash commands (`/start`, `/handover`, `/consilium`) — превращены в текст
  "когда юзер пишет 'start' — выполни вот эту последовательность".
- TaskCreate/TaskUpdate UI — у Codex нет встроенного task tracking.
- Frontmatter `paths:` conditional rule loading — нет в Codex, всё всегда on.

**Главное наблюдение из smoke-теста:** на `codex exec` с явным запросом
на memory_stats Codex (gpt-5.5 на тот момент был ещё не сконфигурирован)
**пошёл напрямую в `sqlite3` через bash вместо MCP-tool'а.** Sandbox его
отбил, потом он попробовал python3 heredoc — тоже отбили. Это в точности тот
soft-enforcement gap, о котором я предупреждал в design-discussion'е. Это не
баг моста — мост работает (audit-лог показывает мои собственные тестовые
вызовы с returncode=0, duration 53-60ms). Это разница в training reward
между OpenAI и Anthropic: Codex с большей вероятностью bypasses инструкции
ради autonomy.
