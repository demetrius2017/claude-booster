## Summary

Дмитрий заметил паттерн: после 100-200k токенов в сессии Claude перестаёт делегировать через paired Worker+Verifier и начинает писать код сам. Задача сессии — диагностировать корневую причину и внести минимальный fix.

**Диагноз (2 Explore агента):**
1. Это НЕ compaction (при 1M контексте compaction на 800k, а drift на 100-200k — далеко до порога)
2. Это attention decay трансформера — правила на позиции ~8000 вытесняются 90k+ свежих tool results
3. Единственный refreshed signal на каждом prompt — `phase_prompt_inject.py` — говорил "code edits allowed", фактически reinforcing inline editing
4. `delegate_gate.py` блокирует только 2-й Edit подряд (reactive), но не напоминает (not proactive)
5. PAL MCP calls не считаются delegation signal (docstring обещает, код не делает) — gap для будущего fix'а

**Fix:** Переформулирован IMPLEMENT hint с "code edits allowed; run tests after" на "code edits via delegated agents; run tests after. Lead: delegate coding via Agent (paired Worker+Verifier), budget=N direct action per delegation window." Budget читается из того же env var что и delegate_gate.

**Почему это должно помочь:** phase_prompt_inject — единственный hook который fires на КАЖДОМ UserPromptSubmit. Это единственный signal который Claude видит fresh на каждом шаге, независимо от глубины контекста. Раньше этот signal reinforced проблему; теперь он counter-acts её.
