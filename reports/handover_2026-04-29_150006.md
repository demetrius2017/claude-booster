# Handover — 2026-04-29 15:00

## Summary
Сконфигурировал глобальный auto-approval для Codex CLI в `~/.codex/config.toml`,
чтобы он перестал спрашивать разрешения на каждое действие. Использована штатная
модель прав Codex: `approval_policy = "on-failure"` + `sandbox_mode = "workspace-write"`
с включённым `network_access = true`. Внутри workspace codex действует молча
(включая `rm`); попытка выйти за песочницу (системные пути, чужие проекты) —
блокируется sandbox'ом, тогда спрашивается эскалация.

Важный нюанс: у Codex нет per-command-type фильтра «спроси перед `rm`». Поэтому
точный запрос «всё кроме удаления» неосуществим как есть; ближайшая разумная
семантика — «всё внутри workspace без вопросов, всё вне workspace — через
эскалацию». Защита от случайного `rm -rf ~` обеспечивается границей sandbox'а,
не фильтром по команде. Внутренняя страховка — git.
