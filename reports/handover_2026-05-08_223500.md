## Summary

Дмитрий заметил, что Claude хорошо делает консилиумы (multi-agent debate), но плохо аудиты — фактически делает аудит сам, один, без разносторонних BIO, без независимого RECON с разных сторон, без независимых подитогов.

**Диагноз:** Consilium и audit были описаны одним блоком в `consilium.md` — разница ровно одна строка в step 6 (куда сохранять файл). Слово "audit" триггерит другой mental model ("один ревьюер проверяет"), и Claude сваливался в single-agent. Инструкция была слишком слабой, чтобы перебить эту интуицию.

**Решение:** Выделен `/audit` в полноценную отдельную команду (751 строк после /simplify) с:
- 6 audit lenses (Correctness, Security, Performance, Architecture, Data Integrity, Operational) — каждый с BIO-параграфом и обязательными grep-паттернами
- Параллельный спавн всех аудиторов + PAL в ОДНОМ сообщении
- Структурированные вердикты (PASS/FAIL/CONCERN) с file:line + code evidence
- Report template с verdict matrix, cross-lens findings, prioritized action items
- Argument parsing: `--scope` для ограничения файлов, `--focus` для выбора линз

**Consilium.md очищен:** убран "or audit" из описания и audit output path из step 6.
