## What was done

### 1. stream_json_adapter.py — Path A WorkerRuntime — 255 LOC
`templates/scripts/supervisor/stream_json_adapter.py`.

- Spawns `claude -p --output-format stream-json --verbose` as an
  asyncio subprocess; `stdin=DEVNULL`, `stderr=DEVNULL` (fix for
  audit-finding: piped-but-undrained stderr deadlocks the CLI when
  diagnostics exceed the pipe buffer).
- `_reader` consumes stdout line-by-line, enforces schema handshake
  on the first parseable JSON line, and emits normalised
  `WorkerEvent`s into a per-task `asyncio.Queue`. Sentinel `None`
  written exclusively in `_reader.finally` — `cancel()` no longer
  races it (audit-finding M-cancel-race).
- `_to_events(state, msg) -> list[WorkerEvent]`: emits **every**
  content block in an assistant message, not just the first match.
  The pre-audit code returned on the first `text`/`tool_use`/
  `thinking` block and silently dropped the rest — which would have
  let a bundled `tool_use` slip past policy enforcement entirely
  (audit-finding H-multi-block).
- `_reader` hard-fails if the first non-empty line is invalid JSON,
  closing the non-JSON-garbage bypass of the `system/init` handshake
  (audit-finding M-schema-bypass).
- `cancel()` signal ladder SIGINT → 2 s → SIGTERM → 2 s → SIGKILL,
  swallowing only `ProcessLookupError`.
- `shutdown()` narrowed `except` to `asyncio.TimeoutError` +
  `SchemaMismatchError`; no longer swallows `CancelledError`.
- Docstring now explicitly flags that `submit_task(cwd=...)` performs
  no path-escape validation (audit-finding L-cwd-doc).

### 2. detector.py — adaptive silence FSM — 158 LOC
`templates/scripts/supervisor/detector.py`.

- State machine over the 9 states consilium §5/Q3 mandated:
  `queued`, `running`, `thinking`, `waiting_on_tool`,
  `possibly_complete`, `completed`, `failed`, `cancelled`,
  `blocked_by_quota`.
- Adaptive silence threshold = `clamp(3 × median(gaps), 20 s, 180 s)`.
  Median seeded after ≥ 3 inter-event gaps; before that the 60 s
  grace-window after `message_start` keeps `possibly_complete` from
  firing. Compound guard: `(now-started < grace) AND (gaps < 3)` →
  skip; either condition breaking allows the check.
- Text accelerator (`should i`, `can i`, `would you like`, `let me
  know`, `please confirm`, `waiting for`) halves the silence
  threshold but **never** flips the FSM by itself — state only moves
  via an event or a `tick()`. Explicit regression test:
  `test_accelerator_never_authoritative_by_itself`.
- `force(state)` for external terminal overrides (quota circuit
  `open`, supervisor cancel) — validates that only terminal states
  are allowed.

## Problems / Solutions

### P1: GPT codereview surfaced H-stderr deadlock I'd missed
Piping `stderr=subprocess.PIPE` without a drain task can block the
CLI if it writes past the pipe buffer (~64 KB on macOS, ~4 KB on
some Linux). Fix: `stderr=DEVNULL`. Supervisor will never surface
worker stderr in MVP — if Session 4 needs it, add a dedicated drain
task plus shutdown-await/cancel.

### P2: cancel() + _reader both pushing None raced the consumer
Pre-audit `cancel()` put `None` into `state.queue` AND `_reader` did
the same in `finally`. A fast cancel could close the queue before
`_reader` enqueued buffered events, truncating `tool_use_stop`s and
the final `message_stop` usage snapshot. Fix: `cancel()` only sets
`state.terminal = "cancelled"`; `_reader.finally` is the single
sentinel writer.

### P3: Non-JSON first line bypassed system/init handshake
`_reader` swallowed `JSONDecodeError` unconditionally. A hostile
wrapper emitting `\n\ngarbage\n{"type":"assistant",...}` would skip
past the schema check, leaving the supervisor with no stream-json
version guarantee. Fix: if `schema_checked` is still False, hard-
raise `SchemaMismatchError` on the first unparsable line.

### P4: _to_event dropped content blocks after the first match
Bundled `text + tool_use` messages (very common in Claude output)
emitted only the `text_delta` — the `tool_use` block was silently
discarded, `state.tool_calls.append(block)` never fired, and the
downstream policy/quota check got no signal. This weakens every
red-team guarantee on the adapter. Fix: `_to_events(...)` returns a
`list[WorkerEvent]`, `_reader` loops over it. Two regression tests
added (unit + reader-replay end-to-end).

### P5: shutdown() caught CancelledError alongside Exception
Generic `except (TimeoutError, Exception)` swallowed legitimate
cancellation bubbling up from the event loop. Fix: split the
handler into `except asyncio.TimeoutError` (soft cancel reader) and
`except SchemaMismatchError` (already reflected in terminal); let
everything else propagate.
