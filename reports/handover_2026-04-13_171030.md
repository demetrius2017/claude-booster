## The Phase 3 detour (worth documenting)

First attempt landed as `11d41a4` on wishful thinking — I renamed
`CLAUDE.md` and claimed success based on Dmitry's fresh session being
able to list rules. Dmitry immediately challenged it: "но было ощущение
что он их загружал только после вопроса" (felt like loaded reactively,
not at session start). I rolled back in seconds (`34ab124`), restored
`CLAUDE.md`, and proposed reading the actual docs.

Reading `https://code.claude.com/docs/en/memory` gave the authoritative
answer:

> "User-level rules in `~/.claude/rules/` apply to every project on
>  your machine. … Rules without `paths` frontmatter are loaded at
>  launch with the same priority as `.claude/CLAUDE.md`."

So rules/ auto-load IS documented behavior, independent of `CLAUDE.md`.
My original hypothesis was correct; my reasoning "harness parses the
CLAUDE.md bullet list as a manifest" was wrong.

Second attempt (`0dbe377`) used **`/memory` as the ground truth** —
the docs specifically call it out as the debug tool: "The `/memory`
command lists all CLAUDE.md, CLAUDE.local.md, and rules files loaded
in your current session." Also wired an `InstructionsLoaded` hook
(`~/.claude/scripts/instructions_loaded_log.py`) as a second data
source — but the event didn't fire in Claude Code v2.1.104 (docs
describe a newer version). Hook entry removed from settings.json;
script stays on disk for future re-enable.

**Institutional lesson**: Dmitry's observation "felt like loaded
after the question" has a clean technical explanation — LLMs can't
reliably recite long files verbatim from the system prompt, so when
asked "what rules are loaded", Claude runs `Read` against `rules/*.md`
to quote them precisely. That Read is visible in the transcript and
LOOKS like lazy loading, but the content is already in context. The
Read is for citation fidelity, not for discovery. This pattern will
recur any time we debug "is X loaded at session start?". The answer:
**use `/memory` or `InstructionsLoaded` hook, not Claude's own
introspection**.

Promoting to `institutional.md` as a Claude Code / Tooling rule after
this handover commits.
