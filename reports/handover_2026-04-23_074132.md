## Проблемы и решения (по ходу эксперимента)

### Проблема 1: v1 показал нулевую дельту на Sonnet 4.6 и Opus 4.7
**Причина:** text injection `[state: vigilance=47]` в system-reminder модель игнорирует — не меняет поведение.
**Решение:** v3 — крутить реальные ручки API (`effort`, `max_tokens`, `task_budget`) через controller, а не инъектировать текст.

### Проблема 2: retry_grind_recon_switch не дал дельты даже в v3
**Причина:** Opus 4.7 слишком competent, всегда сразу грепит, игнорируя memory path.
**Решение:** добавить **force-memory gate** — на scenario с `memory_only_first_turn: True` первый turn физически ограничен tools=[MEMORY_TOOL_SCHEMA]. Форсит модель пройти через memory → тест invalidation path становится возможным.

### Проблема 3: escalation_latency text-injection могла быть проигнорирована
**Причина:** умная модель может проигнорировать "stop and escalate" в system prompt.
**Решение:** **EXHAUSTED hard gate** — harness сам прерывает сценарий, `effective_tools = []`. Модель физически не может продолжать.

### Проблема 4: correctness regex ненадёжен
**Причина:** regex `doesn'?t exist` не матчится на `does not exist` (apostrophe vs space).
**Решение:** задокументировано как минус метрики; для production — использовать LLM-judge для correctness check'ов.

### Проблема 5: OFF arm в memory_staleness застрял в "memory-only mindset" после T0
**Причина:** даже когда T1 tools открыты, модель следует контексту T0 и отказывается использовать grep/read.
**Вывод:** это **дополнительная валидация принципа "гейты важнее инструкций"** — в двусторонне направлении. Инъекция "у тебя full tools" не помогла бы.
